## Block 2 — DSA (35 min)
### Two Sum — Arrays + Hash Map

**NeetCode:** https://neetcode.io/problems/two-sum  
**Difficulty:** Easy  
**Category:** Arrays

**Problem:**  
Given an array `nums` and integer `target`, return indices `[i, j]` such that `nums[i] + nums[j] == target`. Exactly one solution exists. Cannot use same element twice.

---

### Approach 1 — Brute Force (understand first)

```python
def two_sum_brute(nums, target):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
```

**Time:** O(n²) | **Space:** O(1)  
Too slow for n = 10⁴.

---

### Approach 2 — Hash Map (optimal)

**Intuition:** For each number, the complement you need is `target - num`. Ask: "have I seen the complement already?" A dict answers that in O(1).

```python
def two_sum(nums, target):
    seen = {}  # value -> index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
```

**Time:** O(n) | **Space:** O(n)

**Walk through `[2, 7, 11, 15]`, target `9`:**
```
i=0: num=2, complement=7, seen={} → miss → seen={2:0}
i=1: num=7, complement=2, seen={2:0} → HIT → return [0, 1]
```

**Solution file:** `dsa/arrays/01_two_sum.py`

## Combined Task — `/two-sum` Endpoint

Add to `fastapi/01_hello_world/main.py`:

```python
from pydantic import BaseModel
from typing import List

class TwoSumRequest(BaseModel):
    nums: List[int]
    target: int

@app.post("/two-sum")
def solve_two_sum(request: TwoSumRequest):
    seen = {}
    for i, num in enumerate(request.nums):
        complement = request.target - num
        if complement in seen:
            return {"indices": [seen[complement], i], "found": True}
        seen[num] = i
    return {"indices": [], "found": False}
```

**Test in Swagger UI (`/docs`):**
```json
{ "nums": [2, 7, 11, 15], "target": 9 }
```
Expected: `{ "indices": [0, 1], "found": true }`

**What this demonstrates:**
- Python `dict` (Block 1) → used as hash map (Block 2) → exposed as API (Block 3)
- `BaseModel` for request validation — if you send wrong types, FastAPI rejects with 422 automatically
- Type hints in Python (`List[int]`) power the auto-validation
