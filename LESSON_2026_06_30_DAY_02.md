## 🐍 PYTHON — Strings: Slicing, Methods & Immutability

### Intuition

A Python `str` is an immutable sequence of Unicode characters. "Immutable" means once created, it never changes — every method that looks like it modifies a string (`.upper()`, `.replace()`, `.strip()`) actually returns a brand-new string and leaves the original untouched. This matters because it's the opposite of how lists behave, and confusing the two causes real bugs later.

The second big idea is **slicing**: `s[start:stop:step]`. It's not a special string feature — it works on lists, tuples, and any sequence — but strings are where you'll use it most, especially for reversal (`s[::-1]`) and substrings.

### The Essentials

```python
word = "Sprint"

# Indexing — 0-based, negative indices count from the end
word[0]    # 'S'
word[-1]   # 't'

# Slicing: [start:stop:step] — stop is EXCLUSIVE
s = "abcdefgh"
s[2:5]     # 'cde'   — chars at index 2, 3, 4
s[:3]      # 'abc'   — from the start
s[3:]      # 'defgh' — to the end
s[::2]     # 'aceg'  — every 2nd character
s[::-1]    # 'hgfedcba' — step -1 walks backward: the classic reverse trick

# Methods return NEW strings — they never mutate in place
"  hi  ".strip()              # 'hi'
"Hello".lower()                # 'hello'
"a,b,c".split(",")             # ['a', 'b', 'c']
"-".join(["a", "b", "c"])      # 'a-b-c'
"Hello World".replace("World", "Sprint")  # 'Hello Sprint'
```

### Counting characters — the pattern you'll reuse all day

```python
# Same dict-counting idea from Day 1, applied to characters
freq = {}
for ch in "anagram":
    freq[ch] = freq.get(ch, 0) + 1
# {'a': 3, 'n': 1, 'g': 1, 'r': 1, 'm': 1}

# Alternative: sort the characters and compare
"".join(sorted("anagram"))   # 'aaagmnr' — useful for comparing two strings
```

### Why it matters going forward

Any problem phrased as "do these two things use the same letters/elements?" reduces to either (a) sort both and compare, or (b) count both into dicts and compare. You're about to use option (b) to solve today's DSA problem in one pass.

## 🧩 DSA — Valid Anagram *(Easy · Strings)*

**NeetCode:** https://neetcode.io/problems/valid-anagram

### The Problem

Given two strings `s` and `t`, return `True` if `t` is an anagram of `s` — same characters, same frequency, different (or same) order — and `False` otherwise.

### Reading the problem like an engineer

Strip away the word "anagram" and restate it: *"Do these two strings contain exactly the same multiset of characters?"* That's a counting problem. Two strings have the same multiset of characters if and only if every character appears the same number of times in both.

### Approach 1 — Sorting (write this first, always)

```python
def is_anagram_sort(s: str, t: str) -> bool:
    if len(s) != len(t):
        return False
    return sorted(s) == sorted(t)
    # Time:  O(n log n) — dominated by the sort
    # Space: O(n) — sorted() builds new lists
```

Simple and correct. Good enough for most cases, but an interviewer will ask you to do better.

### Approach 2 — Hash Map / Frequency Count (the insight)

```python
def is_anagram(s: str, t: str) -> bool:
    if len(s) != len(t):
        return False

    counts = {}
    for ch in s:
        counts[ch] = counts.get(ch, 0) + 1

    for ch in t:
        if ch not in counts:
            return False        # t has a char s never had
        counts[ch] -= 1
        if counts[ch] == 0:
            del counts[ch]       # fully matched — clean it up

    return len(counts) == 0      # nothing left over means a perfect match

    # Time:  O(n) — two linear passes
    # Space: O(1) — bounded by alphabet size (26 for lowercase English)
```

### Walkthrough on `s = "anagram"`, `t = "nagaram"`

```
Build counts from s: {'a': 3, 'n': 1, 'g': 1, 'r': 1, 'm': 1}

Walk t, decrementing:
 n -> counts={'a':3,'n':0→del,'g':1,'r':1,'m':1}
 a -> counts={'a':2,'g':1,'r':1,'m':1}
 g -> counts={'a':2,'g':0→del,'r':1,'m':1}
 a -> counts={'a':1,'r':1,'m':1}
 r -> counts={'a':1,'r':0→del,'m':1}
 a -> counts={'a':0→del,'m':1}
 m -> counts={'m':0→del}  -> counts={}

len(counts) == 0  ->  True ✓
```

### Why check `len(s) != len(t)` first?

Without it, `"aa"` vs `"a"` would still pass the frequency check for the characters present in `t`, even though `s` has an extra leftover `'a'` the loop never inspects. The length check is a cheap O(1) guard that eliminates a whole class of false positives.

## ⚡ FASTAPI — Path & Query Parameters

### Intuition

You already used path parameters on Day 1 (`/hello/{name}`). Today's new piece is **query parameters** — anything that follows `?` in a URL, like `/search?q=hello&limit=5`. FastAPI's rule is simple: any function parameter that *isn't* named in the path string is automatically treated as a query parameter.

### The Essentials

```python
from fastapi import FastAPI
from typing import Optional

app = FastAPI(title="Sprint Day 2")


# {name} is in the path -> path parameter, required
# limit is NOT in the path -> query parameter
# Giving it a default (= 10) makes it OPTIONAL
@app.get("/search")
def search(q: str = "", limit: int = 10):
    return {"query": q, "limit": limit}

# GET /search                  -> q="", limit=10
# GET /search?q=hi              -> q="hi", limit=10
# GET /search?q=hi&limit=3       -> q="hi", limit=3


# Optional[bool] with a default — FastAPI coerces "true"/"false"/"1"/"0" automatically
@app.get("/greet/{name}")
def greet(name: str, loud: Optional[bool] = False):
    greeting = f"Hi, {name}!"
    return {"greeting": greeting.upper() if loud else greeting}

# GET /greet/Hasnat?loud=true  -> "HI, HASNAT!"
```

### Required vs optional — the rule that trips everyone up

A query parameter with **no default value** is required, exactly like a path parameter — FastAPI returns `422` if it's missing. Give it a default (even `None`) to make it optional.

```python
@app.get("/check-anagram")
def check_anagram(s: str, t: str):
    # Both s and t have NO default -> both REQUIRED query params
    # Missing either one -> 422 before this function ever runs
    ...
```

Visit **`/docs`** and try `/check-anagram` without the `t` parameter — watch FastAPI reject it automatically.

## 🔗 COMBINED TASK — Wiring All Three

The frequency-count dict from Python becomes the Valid Anagram algorithm, which becomes the logic behind a query-parameter endpoint. Same dict, three layers, one more time.

```python
# main.py — Day 2 combined endpoint

@app.get("/check-anagram")
def check_anagram(s: str, t: str):
    """
    GET /check-anagram?s=anagram&t=nagaram
    -> { "s": "anagram", "t": "nagaram", "is_anagram": true }

    Identical algorithm to the DSA block — the only new concept is that
    s and t arrive as validated query-string values instead of function args.
    """
    if len(s) != len(t):
        return {"s": s, "t": t, "is_anagram": False}

    counts = {}
    for ch in s:
        counts[ch] = counts.get(ch, 0) + 1
    for ch in t:
        if ch not in counts:
            return {"s": s, "t": t, "is_anagram": False}
        counts[ch] -= 1
        if counts[ch] == 0:
            del counts[ch]

    return {"s": s, "t": t, "is_anagram": len(counts) == 0}
```

## 🔑 CONCEPT ANSWER

**Question:** "Why is `s += "x"` inside a loop considered an anti-pattern in Python, even though it looks O(1) per iteration?"

**Answer:** Strings are immutable, so `s += "x"` doesn't append in place — it builds an entirely new string object containing the old contents plus `"x"`, then rebinds `s` to it. Each iteration copies everything that came before, so concatenating `n` times costs `1 + 2 + 3 + ... + n` character copies — O(n²) total, not O(n). The fix is to accumulate pieces in a list and call `"".join(parts)` once at the end, which allocates the final string exactly once in O(n). CPython has a micro-optimization that sometimes makes `+=` on strings fast in a tight loop, but it's an implementation detail, not a language guarantee — don't rely on it, and don't write it that way in an interview.
